# 簿記3級 学習アプリ 設計書

## 1. 全体構成

本アプリは日商簿記3級の試験形式に沿って設計し、**本番と同じ形式**で学習できることを目指す。

---

## 2. 第1問：仕訳（現状と強化）

### 現状
- ✅ 約120パターン × 金額バリエーション = 3,000問以上
- ✅ 15問セットで本番形式
- ✅ 借方・貸方の入力、採点、解説
- ✅ カテゴリ別出題可能

### 強化ポイント
| 項目 | 内容 |
|------|------|
| **本番モード** | 15問を時間制限（目安15分）で解く模擬試験モード |
| **カテゴリ選択** | 苦手なカテゴリに絞って練習できる |
| **復習機能** | 不正解だった問題を再度出題 |
| **進捗の可視化** | カテゴリ別の正答率表示 |

### データ構造（現状維持）
```
JournalEntryTemplate: id, question, debit, credit, explanation, category
→ 金額バリエーションで問題生成
```

## 3. 第2問：補助簿・勘定記入・用語選択（新規開発）

### 本番の出題形式

第2問は**問1**と**問2**の2つに分かれる（計20点）。

| 出題タイプ | 形式 | 実装方針 |
|------------|------|----------|
| **勘定記入** | T字勘定の空欄を埋める | 取引データ→T勘定表示→入力欄 |
| **補助簿記入** | 現金出納帳・仕入帳・売上帳などに記入 | 表形式UI、取引リスト |
| **伝票記入** | 3伝票制で起票 | 伝票フォーマット、取引選択 |
| **用語選択** | 穴埋め・4択 | クイズ形式 |

---

### 3-1. 用語選択（実装難易度：低）

**本番形式**: 文章の空欄に当てはまる用語を4択から選ぶ

**データ構造**:
```typescript
interface TermQuestion {
  id: string
  question: string      // "借方には（　）の増加が記入される。"
  blankPosition: number // 空欄の位置
  options: string[]     // ["資産", "負債", "純資産", "収益"]
  correctAnswer: string
  explanation: string
  category: string
}
```

**UI**: 4つのボタン、正誤表示、解説

---

### 3-2. 勘定記入（実装難易度：中）

**本番形式**: 複数取引が与えられ、特定の勘定（例：現金、買掛金）のT字勘定の空欄を埋める

**データ構造**:
```typescript
interface AccountEntryProblem {
  id: string
  accountName: string   // "現金" など
  transactions: Array<{ // 取引リスト
    date: string
    description: string
    debit?: number      // 借方金額
    credit?: number     // 貸方金額
  }>
  // 解答：T勘定の各行（日付、摘要、借方、貸方）
  answer: Array<{
    date: string
    description: string
    debit: number | null
    credit: number | null
  }>
}
```

**UI**: 
- 左：取引一覧（問題文）
- 右：T字勘定の表（入力欄付き）
- 本番同様のレイアウト

---

### 3-3. 補助簿記入（実装難易度：中〜高）

**本番形式**: 取引から現金出納帳・仕入帳・売上帳などに記入する

#### 現金出納帳の形式
| 日付 | 摘要 | 収入 | 支出 | 残高 |
|------|------|------|------|------|
| 4/1 | 前月繰越 | | | 50,000 |
| 4/5 | 売上 | 30,000 | | 80,000 |
| ... | | | | |

**データ構造**:
```typescript
interface CashBookProblem {
  id: string
  openingBalance: number  // 前月繰越
  transactions: Array<{
    date: string
    description: string
    type: 'income' | 'expense'
    amount: number
  }>
  answer: Array<{
    date: string
    description: string
    income: number | null
    expense: number | null
    balance: number
  }>
}
```

**同様に**: 仕入帳、売上帳、商品有高帳など

---

### 3-4. 伝票記入（実装難易度：中）

**本番形式**: 3伝票制（入金伝票・出金伝票・振替伝票）に取引を起票

**データ構造**: 仕訳データを流用し、伝票種別の判定ロジックを追加
